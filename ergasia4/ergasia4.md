## Εργαστηριακή Άσκηση 4 — Σημειώσεις και Απαιτήσεις

Προσωπικές μου σημειώσεις για την παρουσίαση της άσκησης από τον διδάσκοντα για την Εργαστηριακή Άσκηση 4. Χρησιμοποιήστε αυτό το έγγραφο ως αναφορά κατά την υλοποίηση του `client`.

### 1. Διαδικαστικά / Επικαιρότητα
- Εξέταση εργαστηρίου: **20 Μαΐου** (όπως αναφέρθηκε στην παρουσίαση).
- Ερωτήσεις/υποστήριξη: χρησιμοποιήστε το αντίστοιχο κανάλι του μαθήματος στο Slack.

### 2. Στόχος της Άσκησης
Υλοποίηση προγράμματος-πελάτη (`client`) που συνδέεται με τον server των διδασκόντων μέσω TCP/IP (sockets) και:
- Λαμβάνει μετρήσεις αισθητήρων (π.χ. θερμοκρασία).
- Υποβάλλει αιτήματα για άδειες μετακίνησης/εξόδου και διαχειρίζεται την πρωτοκόλληση επιβεβαίωσης.

### 3. Βασικά Θέματα / Θεωρητικό Υπόβαθρο
- Domain: Χρησιμοποιούμε το Internet domain (`AF_INET`) — όχι Unix-domain sockets.
- Τύπος σύνδεσης: Stream sockets (`SOCK_STREAM`) — πρωτόκολλο TCP.
- Τα sockets ταυτοποιούνται με `IP:PORT`.

### 4. Παράμετροι Εκτέλεσης
Το πρόγραμμα πρέπει να αποδέχεται τις παρακάτω παραμέτρους γραμμής εντολών:

```
./ask4 [--ip IP] [--port PORT] [--debug]
```

- Προεπιλεγμένες τιμές: `IP=147.102.75.201`, `PORT=4217`.
- Σημείωση: δεν επιτρέπεται η χρήση hostnames — μόνο διευθύνσεις IPv4 ως string.

### 5. Πρωτόκολλο Εφαρμογής — Υποστηριζόμενες Εντολές
Ο `client` πρέπει να διαβάζει εντολές από το τερματικό και να επικοινωνεί με τον server σύμφωνα με την παρακάτω διεπαφή:

- `help`
	- Εμφανίζει βοήθεια/συντομεύσεις.

- `exit`
	- Κλείνει ομαλά τη σύνδεση και τερματίζει τον client.

- `get`
	- Ζητάει από τον server την τρέχουσα μέτρηση αισθητήρα.
	- Ο server επιστρέφει συμβολοσειρά: `X YYY ZZZZ TTTTTTTT` όπου:
		- `X` = τύπος συμβάντος (π.χ. 0: boot, 2: interval, 4: motion)
		- `YYY` = επίπεδο φωτεινότητας
		- `ZZZZ` = θερμοκρασία (πρέπει να διαιρεθεί με 100 για πραγματική τιμή)
		- `TTTTTTTT` = UNIX timestamp (μετατρέψτε με `localtime()` / `strftime()`)

- `N name surname reason`
	- Υποβολή αίτησης άδειας (π.χ. `1 aggelos kolaitis dokimi`).
	- Πρωτόκολλο:
		1. Ο server επιστρέφει έναν κωδικό επιβεβαίωσης.
		2. Ο client διαβάζει τον κωδικό και τον στέλνει πίσω στον server.
		3. Ο server αποκρίνεται με `ACK` (επιτυχία) ή `try again` (αποτυχία — επαναπροσπάθεια).

### 6. Τεχνικές Απαιτήσεις / Περιορισμοί

- I/O Multiplexing: Το πρόγραμμα πρέπει να περιμένει ταυτόχρονα είσοδο από το τερματικό και από το socket — απαιτείται χρήση της system call `poll()`.
- Απαγορεύεται η χρήση `gethostbyname()` ή άλλων συναρτήσεων που επιλύουν hostnames.
	- Πρέπει να μετατρέψετε χειροκίνητα την IPv4 διεύθυνση σε bytes (π.χ. `sscanf()` ή `inet_pton()` εάν επιτρέπεται) και να γεμίσετε σωστά την `struct sockaddr_in`.

### 7. Σημεία Υλοποίησης / Προτάσεις
- Χρήση `poll()`:
	- Επιτηρήστε δύο fd: `STDIN_FILENO` και το socket fd. Όταν υπάρχουν δεδομένα από το socket, διαβάστε και χειριστείτε. Όταν υπάρχουν δεδομένα από το stdin, στείλτε κατάλληλο αίτημα στον server.

- Διαχείριση συμβολοσειρών και parsing:
	- Ελέγξτε τα όρια buffers, χρησιμοποιήστε `snprintf()` / `strtok_r()` και ελέγξτε την εγκυρότητα των τιμών πριν την επεξεργασία.

- Χρόνος αναμονής / επαναπροσπάθεια:
	- Ενσωματώστε timeout για το socket όταν περιμένετε επιβεβαίωση, και λογική επαναπροσπάθειας όταν ο server απαντά `try again`.

### 8. Παραδείγματα Εκτέλεσης

```
./ask4 --ip 147.102.75.201 --port 4217
> help
> get
< 2 512 2350 1653000000
	(θερμοκρασία = 2350 / 100 = 23.50 °C)
> 1 aggelos kolaitis dokimi
< CONFIRM_CODE_12345
> CONFIRM_CODE_12345
< ACK
```

### 9. Έλεγχος / Παράδοση
- Βεβαιωθείτε ότι ο client χειρίζεται σωστά:
	- `poll()` και πολλαπλά fd
	- parsing του μηνύματος `get`
	- πρωτόκολλο επιβεβαίωσης για αιτήματα άδειας
- Παραδώστε τον πηγαίο κώδικα και σύντομο `README` με οδηγίες εκτέλεσης και παραδείγματα.

### 10. Λογική του `client.c`
Ο κώδικας του client ακολουθεί ένα απλό αλλά σωστά δομημένο μοτίβο:

1. **Ανάγνωση παραμέτρων**
	- Διαβάζει `--ip`, `--port` και `--debug` από τη γραμμή εντολών.
	- Αν δεν δοθούν τιμές, χρησιμοποιεί τις προεπιλογές του εργαστηρίου.

2. **Δημιουργία socket και σύνδεση**
	- Δημιουργεί TCP socket με `socket(AF_INET, SOCK_STREAM, 0)`.
	- Συμπληρώνει τη δομή `sockaddr_in` με IP και port.
	- Η IP μετατρέπεται χειροκίνητα από string σε bytes, χωρίς `gethostbyname()`.
	- Καλεί `connect()` για να συνδεθεί στον server.

3. **Κεντρικός βρόχος με `poll()`**
	- Παρακολουθεί ταυτόχρονα δύο πηγές:
	  - το `STDIN_FILENO` για εντολές του χρήστη,
	  - το socket για απαντήσεις από τον server.
	- Έτσι ο client δεν «κολλάει» περιμένοντας μόνο μία από τις δύο πλευρές.

4. **Χειρισμός εντολών από το terminal**
	- `help`: εμφανίζει τις διαθέσιμες εντολές.
	- `exit`: τερματίζει ομαλά το πρόγραμμα.
	- οποιαδήποτε άλλη γραμμή: στέλνεται αυτούσια στον server.

5. **Χειρισμός δεδομένων από τον server**
	- Διαβάζει με `read()` την απάντηση του server.
	- Αν ο server κλείσει τη σύνδεση, ο client τερματίζει.
	- Αν το `debug` είναι ενεργό, τυπώνει επιπλέον πληροφορίες για αποστολές/λήψεις.

6. **Πού χρειάζεται ακόμη επέκταση**
	- Το `get` χρειάζεται parsing της μορφής `X YYY ZZZZ TTTTTTTT` και μετατροπή του timestamp.
	- Το `N name surname reason` χρειάζεται ειδικό χειρισμό επιβεβαίωσης (`ACK` / `try again`).
	- Δηλαδή ο τωρινός client έχει τη βασική υποδομή, αλλά το πρωτόκολλο εφαρμογής θέλει ακόμα εξειδίκευση.

### 11. Πιθανές Ερωτήσεις Καθηγητή
Αυτές είναι ερωτήσεις που είναι πιθανό να τεθούν προφορικά πάνω στον κώδικα ή στις αποφάσεις υλοποίησης:

- Γιατί χρησιμοποιείς `poll()` και όχι `select()`;
- Τι πρόβλημα λύνει το `poll()` στον συγκεκριμένο client;
- Ποια είναι η διαφορά μεταξύ `AF_INET` και `AF_UNIX`;
- Γιατί ο server/client πρέπει να χρησιμοποιούν `SOCK_STREAM`;
- Πώς μετατρέπεται μια IPv4 διεύθυνση από string σε network byte order χωρίς DNS;
- Τι κάνει η `htons()` και γιατί είναι απαραίτητη στο port;
- Τι θα συνέβαινε αν δεν έλεγες το `revents` μετά το `poll()`;
- Γιατί χρειάζεται έλεγχος επιστροφής σε `socket()`, `connect()`, `poll()`, `read()` και `write()`;
- Πώς θα χειριστείς το `get` μήνυμα και πώς θα μετατρέψεις τη θερμοκρασία από ακέραιο σε δεκαδικό;
- Τι σημαίνει ότι το `poll()` παρακολουθεί ταυτόχρονα stdin και socket;
- Πώς θα υποστηρίξεις το πρωτόκολλο επιβεβαίωσης για το αίτημα άδειας;

### 12. Σύντομη Απάντηση-Οδηγός για Προφορική Εξήγηση
- Ο client είναι event-driven: περιμένει γεγονότα από δύο πηγές με `poll()`.
- Η IP δεν επιλύεται με host lookup, αλλά μετατρέπεται χειροκίνητα σε bytes.
- Ο στόχος είναι να μη μπλοκάρει το πρόγραμμα όταν περιμένει τον χρήστη ή τον server.
- Το `debug` βοηθάει στον έλεγχο της ροής, χωρίς να αλλάζει τη λογική του πρωτοκόλλου.
